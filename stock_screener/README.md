# 日线复盘选股器

这是一个独立于 `QMT / xtquant` 的复盘回测工具，适合做：

- 先把历史数据同步到本地
- 指定某一天做条件选股
- 按“次日开盘买入”
- 按“持有 N 个交易日后收盘卖出”
- 对比一周或多日后的收益
- 在 K 线图上复核信号、买点和卖点
- 用键盘上下键快速翻页复盘

## 当前版本

当前版本已经从“逐股 CSV 扫描”升级为：

- `BaoStock` 负责历史数据来源
- `DuckDB` 负责本地数据库缓存
- GUI 继续保留
- 首次会自动把旧的 `stock_screener/cache/daily/*.csv` 迁移到本地数据库
- 后续筛选直接走本地数据库，不再逐股读 CSV

这套结构更适合跑 `沪深所有A股` 这类全市场复盘。

## 支持的条件

当前可按开关组合这些条件：

- `收盘上穿 EMA`
- `量比 >= 阈值`
- `日K角度 >= 阈值`
- `换手率 >= 阈值`

交易假设：

- 买入价：`下一交易日开盘价`
- 卖出价：`持有第 N 个交易日收盘价`

## 推荐流程

1. 先选择股票池和同步日期，点击 `同步数据`
2. 如果你之前已经同步过旧版 CSV，首次筛选时会自动迁移到数据库
3. 设置筛选条件后点击 `开始筛选`
4. 在结果表中点股票，右侧 K 线会标记信号、买点、卖点
5. 用键盘上下键快速翻页

## 启动

在项目根目录运行：

```powershell
.\start_review_screener.bat
```

或者：

```powershell
.\.venv311\Scripts\python -m stock_screener
```

说明：

- 推荐优先使用 `start_review_screener.bat` 启动。
- 当前版本按单实例使用设计，已经打开一个窗口时再次启动会直接提示，不再重复打开。
- 便携 exe 版会把缓存写到 exe 同目录下的 `cache/` 和 `reports/`

## 数据与缓存

- 数据源：`BaoStock`
- 周期：`日线`
- 本地数据库：`stock_screener/cache/market_data.duckdb`
- 旧版 CSV 缓存：`stock_screener/cache/daily`
- 报告目录：`stock_screener/reports`

说明：

- 第一次升级到数据库版时，会自动把旧 CSV 缓存迁移进 `DuckDB`
- 之后筛选直接查本地数据库，速度会明显快很多
- 如果遇到 `用户未登录` 或网络抖动，数据同步会自动重连后重试一次

便携版说明：

- 如果你使用 Windows zip 便携版，数据库路径会变成 exe 同目录下的 `cache/market_data.duckdb`
- 报告会写到 exe 同目录下的 `reports/`
- 解压后的整个目录建议保持在同一个位置，不要只单独移动 exe

## 使用建议

- 做全市场扫描时，数据库版更适合长期使用
- 如果你是“当天选股，第二天手动下单”，主周期优先看 `日线`
- 指数成分股适合快速验证，`沪深所有A股` 更适合真实复盘

## 已知边界

- 指数成分股默认使用当前或最近一次同步到本地的成分列表
- 停牌、上市不久、未来持有区间数据不足的股票会自动跳过
- 同一份 `stock_screener/cache/market_data.duckdb` 缓存数据库请只保持一个实例占用
- 这是一套复盘和思路验证工具，不是实盘自动交易终端
